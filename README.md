# 技术支撑事业部项目管理平台 V1.0

一个基于 `PHP + MySQL` 的轻量项目管理系统，面向技术支撑事业部日常项目跟踪、人员绩效、出勤管理、项目文件管理和报表导出场景。

项目当前适合放在 `phpstudy` 或常规 `Nginx / Apache + PHP + MySQL` 环境中直接运行，不依赖 Composer。

## 功能概览

- 登录与权限管理
  - 管理员
  - 普通用户
  - 审核员
- 数据概览
  - 项目总数
  - 总工时 / 总天数
  - 整体出勤率
  - 区域分布与趋势图表
- 项目管理
  - 新增、编辑、删除、预览
  - 分页展示
  - 批量选择删除
  - 项目字段显隐
  - 工单标签管理
  - 反馈标签管理
- Excel 导入导出
  - 支持项目 Excel 导入
  - 支持项目 Excel 导出
  - 导入字段已与当前导出格式对齐
- 人员绩效
  - 按年 / 月 / 周查看绩效
  - 查看个人每日项目任务
  - 管理员可调整每日绩效
  - 支持一键生成绩效表
  - 支持一键生成个人绩效表
- 出勤管理
  - 自动按项目识别出勤状态
  - 管理员可手动调整为调休
- 项目文件
  - 上传、下载、预览
  - 按关键字搜索
  - 按项目归档
- 统计报表
  - 图表联动
  - 项目导出预览
- 系统设置
  - 图标配置
  - 外观配置
  - 账号管理
- 操作日志
  - 登录、项目变更、人员绩效调整等关键操作留痕

## 技术说明

- 后端：原生 PHP
- 数据库：MySQL
- 前端：HTML + Tailwind CDN + 少量原生 JavaScript
- 图表：Chart.js
- Excel：项目内置 `xlsx` 生成与解析逻辑

## 运行环境

- PHP 7.3 及以上
- MySQL 5.7 及以上
- Web 服务
  - Apache
  - Nginx
  - 或 `phpstudy`

## 本地运行

### 1. 放入站点目录

将项目放到：

```text
F:\phpstudy_pro\WWW\pre
```

### 2. 配置站点

站点根目录指向：

```text
F:\phpstudy_pro\WWW\pre
```

入口文件：

```text
index.php
```

### 3. 配置数据库

数据库配置文件：

[`config/app.php`](F:\phpstudy_pro\WWW\pre\config\app.php)

按你的实际环境修改：

```php
'db' => [
    'host' => '127.0.0.1',
    'port' => 3306,
    'database' => 'your_database',
    'username' => 'your_username',
    'password' => 'your_password',
    'charset' => 'utf8mb4',
    'auto_create_database' => false,
],
```

### 4. 启动访问

浏览器访问：

```text
http://127.0.0.1/pre/index.php
```

## 主要页面说明

### 数据概览

展示项目总数、总工时、总天数、出勤率以及统计图表。

### 项目管理

支持：

- 筛选项目
- 导入项目
- 导出 Excel
- 新增项目
- 项目预览
- 编辑项目
- 删除项目
- 批量删除

### 人员绩效

支持：

- 切换年 / 月 / 周维度
- 查看所有技术人员绩效卡片
- 查看个人绩效日历明细
- 管理员保存每日绩效
- 一键生成整体绩效
- 一键生成个人绩效

### 出勤管理

支持：

- 查看本周出勤台账
- 自动识别空闲 / 有项目
- 管理员手动标记调休

### 项目文件

支持：

- 按项目上传文件
- 搜索文件名 / 说明 / 所属项目
- 下载与在线预览

### 统计报表

支持：

- 项目区域分布
- 月度项目趋势
- 项目导出预览
- 一键导出 Excel

## Excel 说明

### 项目导入

支持：

- `.xlsx`
- `.csv`

导入入口在项目管理页面。

### 项目导出

项目导出会输出标准项目计划表，当前已支持：

- 反馈标签列
- 完成评价【销售反馈简述】列

### 绩效导出

人员绩效页面支持：

- 一键生成整体绩效表
- 一键生成个人绩效表

  - 系统配置
- `database`

  - 业务服务层
- `src/Support`

- `storage`
  - 上传文件、临时文件、日志

## 上传到 Git 仓库前建议
